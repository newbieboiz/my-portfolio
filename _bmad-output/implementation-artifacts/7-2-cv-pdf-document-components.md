## Story

As the **owner**,
I want the CV PDF to be composed of typed `@react-pdf/renderer` components that read from the same JSON data as the website,
so that the PDF and portfolio website are always in sync — one data source, two surfaces.

## Acceptance Criteria

1. **Given** `src/components/cv/CVDocument.tsx` is implemented using `@react-pdf/renderer`
   **When** it renders
   **Then** it reads `Experience[]`, `SkillCategory[]`, `Education[]`, and `SiteConfig` from the shared `data/*.json` files via typed data functions (`getExperience()`, `getSkills()`, `getEducation()`, `getSiteConfig()` from `src/lib/data.ts`); no content is hardcoded in the PDF components

2. **Given** `CVDocument` includes a QR code section
   **When** the PDF renders
   **Then** the QR code image (data URL from `generateQRDataURL` — passed as the `qrDataUrl` prop) is embedded as a `<Image>` element in the PDF footer with size 80pt × 80pt and a `<Text>` label "Scan to view live portfolio"

3. **Given** the PDF is exported
   **When** printed on A4 and US Letter paper sizes
   **Then** the layout does not overflow page boundaries; margins are print-safe (min 15mm / 43pt each side); all content fits within a single page for the current seed data (NFR15)

4. **Given** the CV components are in `src/components/cv/`
   **When** the component tree is inspected
   **Then** CV components (`CVDocument`, `CVHeader`, `CVSection`) are isolated from web UI components; no web-only Tailwind classes or browser APIs are used inside CV components; only `@react-pdf/renderer` primitives are used for layout and display

5. **Given** the data schema is the single source of truth
   **When** `data/experience.json` is updated and a new PDF is exported
   **Then** the new experience entry appears in the PDF with no changes to any CV component (FR15, NFR19)

### Critical Architecture: Why `qrDataUrl` Is a Prop (Not Awaited Inside CVDocument)

`generateQRDataURL` is an `async` function. `@react-pdf/renderer` components are **synchronous React function components** — they cannot `await` at the top level. The solution is to pre-generate the QR data URL BEFORE constructing the `CVDocument` element:

**Story 7.3's `CVDownloadButton` will do this:**

```typescript
// Called inside CVDownloadButton's click handler (client-side, "use client")
const qrDataUrl = await generateQRDataURL(getSiteConfig().siteUrl);
const blob = await pdf(<CVDocument qrDataUrl={qrDataUrl} />).toBlob();
```

**Story 7.2's `CVDocument` receives it as a prop:**

```typescript
export function CVDocument({ qrDataUrl }: { qrDataUrl: string }) {
  const experience = getExperience(); // synchronous
  const education = getEducation(); // synchronous
  const skills = getSkills(); // synchronous
  const site = getSiteConfig(); // synchronous
  // ...
}
```

**Why `getSiteConfig().siteUrl` is the correct URL argument in Story 7.3:**
`getSiteConfig()` in `src/lib/data.ts` line 28 constructs the `siteUrl` as `process.env.NEXT_PUBLIC_SITE_URL ?? 'http://localhost:3000'`. This is the ARC2-compliant pattern. `qr.ts` accepts any `string` — the caller provides the URL.

### `@react-pdf/renderer` v4.5.1 API Reference (INSTALLED VERSION)

**Confirmed exports** (verified via `node -e "require('@react-pdf/renderer')"` on this project):

```
Document, Page, View, Text, Image, StyleSheet, Font, Link, pdf, usePDF,
PDFDownloadLink, PDFViewer, BlobProvider, Svg, ...
```

**Correct import pattern** (named imports from the package root):

```typescript
import {
  Document,
  Page,
  View,
  Text,
  Image,
  StyleSheet,
  Font,
} from "@react-pdf/renderer";
```

**NOT** `import ReactPDF from '@react-pdf/renderer'` — use named imports for tree-shaking.

**`Document` component:**

```tsx
<Document title="BaoBao — CV" author="BaoBao">
  <Page size="A4" style={styles.page}>
    {/* content */}
  </Page>
</Document>
```

`DocumentProps` accepts `title`, `author`, `subject`, `creator`, `keywords`, `language` — all optional strings.

**`Page` component:**

```tsx
<Page size="A4" style={styles.page}>
```

`size` accepts: `"A4"`, `"LETTER"`, `"LEGAL"`, `[width, height]` (in points). Use `"A4"` as primary; Letter is tested manually.

**`View` component:** Equivalent to `<div>` but PDF-only. Supports `style`, `wrap`, `debug`.

**`Text` component:** All text content must be inside `<Text>`. Cannot render raw string children in `<View>` — must wrap in `<Text>`.

**`Image` component:**

```tsx
// src accepts a base64 data URL string directly — confirmed by @react-pdf docs:
// "as well as base64 encoded image strings"
<Image src={qrDataUrl} style={styles.qrImage} />
```

`src` type is `SourceObject` which includes `string`. A `data:image/png;base64,...` URL from `generateQRDataURL` is valid.

**`StyleSheet.create` pattern:**

```typescript
const styles = StyleSheet.create({
  page: {
    /* ... */
  },
  heading: {
    /* ... */
  },
});
// Returns the same typed object — use styles.page, styles.heading, etc.
```

**Critical: `StyleSheet` uses PDF/React-Native style properties, NOT CSS shorthand:**

```typescript
// ❌ WRONG — CSS shorthand doesn't work in react-pdf
{ border: '1pt solid #ccc' }
{ padding: '10pt 20pt' }

// ✅ CORRECT — longhand properties only
{ borderBottomWidth: 1, borderBottomColor: '#2a2a3a', borderBottomStyle: 'solid' }
{ paddingTop: 10, paddingBottom: 10, paddingLeft: 20, paddingRight: 20 }
// OR: shorthand equivalents that react-pdf understands
{ paddingVertical: 10, paddingHorizontal: 20 }  // React-Native style — works!
```

**`Font.register` for custom typeface:**

```typescript
// Called at MODULE LEVEL (outside component function, after imports)
// This registers the font globally — call once, use in any StyleSheet via fontFamily
Font.register({
  family: "Courier", // built-in PDF font — no src needed, no network call
});
// To use it: { fontFamily: 'Courier' } in any StyleSheet entry
```

**Built-in PDF fonts** (no `Font.register()` needed, always available):

- `Helvetica`, `Helvetica-Bold`, `Helvetica-Oblique`, `Helvetica-BoldOblique`
- `Times-Roman`, `Times-Bold`, `Times-Italic`, `Times-BoldItalic`
- `Courier`, `Courier-Bold`, `Courier-Oblique`, `Courier-BoldOblique`
- `Symbol`, `ZapfDingbats`

**Font strategy for this MVP:**
Use `Courier` as the document font family — it is a built-in PDF font (requires no `Font.register()` call, no CDN dependency, no network requests), and it delivers the monospace aesthetic consistent with the web portfolio's Geist Mono identity. Apply `fontFamily: 'Courier'` to the page-level style so all text inherits it, then override specific elements with `fontFamily: 'Courier-Bold'` for headings.

### Page Dimensions & Print-Safe Margin Strategy

**A4 in points:** 595.28pt × 841.89pt  
**US Letter in points:** 612pt × 792pt  
**Letter is 49.89pt shorter** than A4 in portrait mode.

**Minimum safe margin (from AC 3, NFR15):** 15mm = 42.52pt → use **45pt** on all sides.

**Printable area on A4:** 505.28pt wide × 751.89pt tall  
**Printable area on Letter:** 522pt wide × 702pt tall (49pt less vertical space)

**Page style:**

```typescript
const styles = StyleSheet.create({
  page: {
    fontFamily: "Courier",
    fontSize: 9,
    paddingTop: 45,
    paddingBottom: 45,
    paddingLeft: 45,
    paddingRight: 45,
    backgroundColor: "#ffffff", // white — PDF is a print document, not a dark-mode web page
    color: "#1a1a25", // near-black for high print contrast
  },
});
```

**Important: PDF background is WHITE.** The web portfolio is dark (`--bg-primary: #0a0a0f`) but the PDF must be print-friendly. Use white background with dark text. The `--accent: #00dc82` green CAN still be used for section headings to create visual identity continuity without dark backgrounds.

#### `src/components/cv/CVDocument.tsx` (core structure)

```typescript
import { Document, Page, View, Text, Image, StyleSheet } from '@react-pdf/renderer';
import { getExperience, getEducation, getSkills, getSiteConfig } from '@/lib/data';
import type { Experience, Education } from '@/types/experience';
import type { SkillCategory } from '@/types/site';
import { CVHeader } from './CVHeader';
import { CVSection } from './CVSection';

const styles = StyleSheet.create({
  page: {
    fontFamily: 'Courier',
    fontSize: 9,
    paddingTop: 45,
    paddingBottom: 45,
    paddingLeft: 45,
    paddingRight: 45,
    backgroundColor: '#ffffff',
    color: '#1a1a25',
    lineHeight: 1.4,
  },
  // Experience entry
  expEntry: { marginBottom: 8 },
  expHeader: { flexDirection: 'row', justifyContent: 'space-between', marginBottom: 2 },
  expCompany: { fontFamily: 'Courier-Bold', fontSize: 9, color: '#1a1a25' },
  expDates: { fontFamily: 'Courier', fontSize: 8, color: '#6b6b80' },
  expRole: { fontFamily: 'Courier', fontSize: 9, color: '#45455a', marginBottom: 2 },
  expDesc: { fontFamily: 'Courier', fontSize: 8, color: '#45455a', marginBottom: 3 },
  achievement: { fontSize: 8, color: '#45455a', marginLeft: 8, marginBottom: 1 },
  // Skills
  skillRow: { flexDirection: 'row', marginBottom: 3, flexWrap: 'wrap' },
  skillCategory: { fontFamily: 'Courier-Bold', fontSize: 8, color: '#1a1a25', marginRight: 6, width: 80 },
  skillList: { fontFamily: 'Courier', fontSize: 8, color: '#45455a', flex: 1 },
  // Education entry
  eduEntry: { marginBottom: 4 },
  eduHeader: { flexDirection: 'row', justifyContent: 'space-between' },
  eduInstitution: { fontFamily: 'Courier-Bold', fontSize: 9, color: '#1a1a25' },
  eduDates: { fontFamily: 'Courier', fontSize: 8, color: '#6b6b80' },
  eduDegree: { fontFamily: 'Courier', fontSize: 8, color: '#45455a' },
  // QR footer
  qrFooter: {
    flexDirection: 'row',
    alignItems: 'center',
    marginTop: 12,
    paddingTop: 10,
    borderTopWidth: 0.5,
    borderTopColor: '#c8c8d0',
    borderTopStyle: 'solid',
  },
  qrImage: { width: 80, height: 80, marginRight: 10 },
  qrLabel: { fontFamily: 'Courier', fontSize: 8, color: '#6b6b80' },
});

export function CVDocument({ qrDataUrl }: { qrDataUrl: string }) {
  const experience = getExperience();
  const education = getEducation();
  const skills = getSkills();
  const site = getSiteConfig();
  const { owner, social } = site;

  return (
    <Document title={`${owner.name} — CV`} author={owner.name}>
      <Page size="A4" style={styles.page}>
        <CVHeader
          name={owner.name}
          title={owner.title}
          email={owner.email}
          github={social.github}
          linkedin={social.linkedin}
        />

        <CVSection title="Experience">
          {experience.map((exp: Experience, i: number) => (
            <View key={i} style={styles.expEntry}>
              <View style={styles.expHeader}>
                <Text style={styles.expCompany}>{exp.company}</Text>
                <Text style={styles.expDates}>
                  {formatDate(exp.startDate)} — {exp.isCurrent ? 'Present' : formatDate(exp.endDate ?? '')}
                </Text>
              </View>
              <Text style={styles.expRole}>{exp.title}</Text>
              <Text style={styles.expDesc}>{exp.description}</Text>
              {exp.achievements.map((a: string, j: number) => (
                <Text key={j} style={styles.achievement}>• {a}</Text>
              ))}
            </View>
          ))}
        </CVSection>

        <CVSection title="Skills">
          {skills.map((cat: SkillCategory, i: number) => (
            <View key={i} style={styles.skillRow}>
              <Text style={styles.skillCategory}>{cat.category}</Text>
              <Text style={styles.skillList}>{cat.skills.map(s => s.name).join(', ')}</Text>
            </View>
          ))}
        </CVSection>

        <CVSection title="Education">
          {education.map((edu: Education, i: number) => (
            <View key={i} style={styles.eduEntry}>
              <View style={styles.eduHeader}>
                <Text style={styles.eduInstitution}>{edu.institution}</Text>
                <Text style={styles.eduDates}>{formatDate(edu.startDate)} — {formatDate(edu.endDate)}</Text>
              </View>
              <Text style={styles.eduDegree}>{edu.degree}, {edu.field}</Text>
            </View>
          ))}
        </CVSection>

        <View style={styles.qrFooter}>
          <Image src={qrDataUrl} style={styles.qrImage} />
          <Text style={styles.qrLabel}>Scan to view live portfolio</Text>
        </View>
      </Page>
    </Document>
  );
}

// Pure helper — formats "2022-03" → "Mar 2022"
function formatDate(isoMonth: string): string {
  if (!isoMonth) return '';
  const [year, month] = isoMonth.split('-');
  const date = new Date(Number(year), Number(month) - 1, 1);
  return date.toLocaleDateString('en-GB', { month: 'short', year: 'numeric' });
}
```

| Anti-Pattern                                                    | Why It Fails                                                                                |

| `import { default as ReactPDF } from '@react-pdf/renderer'`     | Use named imports; default import loses tree-shaking                                        |

### `@react-pdf/renderer` TypeScript Gotchas

1. **`StyleSheet.create<T extends Styles>()` return type** — `Styles` is `{ [key: string]: Style }` from `@react-pdf/types`. The `Style` type accepts a subset of CSS properties. If a property causes a TypeScript error, check the `@react-pdf/types` definition — some CSS shorthand properties are not supported.

2. **`ReactNode` in PDF components** — `children: ReactNode` works for TypeScript, but only `@react-pdf/renderer` primitive elements should be passed as children at runtime. Do NOT pass React DOM elements as children.

3. **`key` prop in mapped elements** — `@react-pdf/renderer` supports React key props. Always include `key={i}` on mapped PDF elements to avoid React reconciliation warnings.

4. **`Image` `src` prop accepts a `SourceObject`** — which includes `string`. A `data:image/png;base64,...` data URL from `generateQRDataURL` is valid. TypeScript will infer this correctly.

5. **`fontFamily: 'Courier-Bold'` vs `fontWeight: 'bold'`** — For built-in PDF fonts, use the explicit bold variant name (`Courier-Bold`) rather than setting `fontWeight: 'bold'` on `Courier`, since `@react-pdf/renderer` resolves bold by looking for a registered bold variant. Without `Font.register()`, the explicit font name is the reliable path.
